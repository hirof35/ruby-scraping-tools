# TkNewsScraper

Ruby 3.4+ と Ruby/Tk を用いて構築された、Windows環境で100%の安定稼働を誇る汎用デスクトップ・ウェブスクレイピングツールです。
不安定なGUIイベントループとOSコマンド（システムコール）の競合を完全に克服し、軽量かつ堅牢なニュース・価格情報の自動収集を実現します。

## 🎯 特徴

* **絶対的な堅牢性（Rock-Solid Stability）**
  C言語レイヤーのメモリ衝突（アクセスバイオレーション）を誘発しやすい不安定なGUIイベント（LibUI等）を徹底排除。実績のある `Tcl/Tk` ツールキットを採用し、Windows 11環境下での連続稼働でもプロセスが強制終了（クラッシュ）しません。
* **強力なスクレイピングバックエンド**
  `Nokogiri` による高速なHTMLパース。User-Agent偽装により、プログラムアクセスに対するサーバー側の403ブロックを回避します。
* **ワンクリック・ブラウザ連動**
  抽出されたリストから対象行を選択し、物理ボタンを介して安全に既定のブラウザ（Edge, Chrome等）を非同期で叩き起こします。
* **相対パスの自動クレンジング**
  対象サイトのHTML内に埋め込まれた相対URL（`/articles/...`等）を、基点URLをベースに絶対URL（`https://...`）へ自動補正します。

## 🛠️ 動作環境 & 必要要件

* **OS:** Windows 10 / 11 (x64)
* **Language:** Ruby 3.4.0 以降
* **GUI Engine:** Tcl/Tk (ActiveTcl または RubyInstaller組み込みの MSYS2 依存環境)

### 依存Gemのインストール
ターミナル（PowerShell等）を開き、以下のコマンドを実行してください。

```bash
gem install tk nokogiri
⚠️ Windows環境での注意点
gem install tk でエラーが発生する場合、Rubyインストール時に同梱されている MSYS2 の開発ツール（ビルド環境）が必要です。PowerShellで ridk install を実行し、3 (MSYS2 system update) を実行してから再試行してください。

🚀 使い方
1. アプリケーションの起動
リポジトリのディレクトリで以下のコマンドを実行し、GUIウィンドウを立ち上げます。

Bash
ruby tool.rb
2. データ抽出の手順
対象URL: 抽出したいウェブサイトのURLを入力します（初期値として Yahoo!ニュース が設定されています）。

抽出の実行: [最新ニュースを抽出] ボタンをクリックします。裏で非同期通信が走り、ページ内の有効なアンカータグ（<a>）からテキストとリンクを根こそぎパースします。

ブラウザでの閲覧: 画面下のリストボックスから、読みたいニュースを1行クリックして選択状態にし、[🌐 選択したニュースを開く] ボタンを押すと、安全にブラウザが起動します。

🔍 テクニカル・アーキテクチャ（構造の解説）
本ツールは開発初期、glimmer-dsl-libui（C言語ベースの軽量ラッパー）を採用していましたが、Windows環境特有のメモリ管理およびGUIメインスレッド内でのシステムコール衝突により、例外コード 0xc0000005（アクセス違反による強制終了）が発生する課題を抱えていました。

現在のバージョンでは、以下の防御的 Thought Engineering（思考工学）に基づき設計が刷新されています。

[User Action] ──> 行を選択 (メモリ保持のみ)
                       │
                       ▼
[Open Button] ──> 単発発火 (GUIスレッドをロックしない)
                       │
                       ▼
[Windows OS]  ──> spawn/system("start") を安全に実行（プロセス分断）
イベント駆動の安全性: リストを「カチカチと連続クリック」した際に発生するメッセージバーストを完全に無効化。選択状態の取得をボタンのトリガー（単発発火）に集約し、OS側のプロセスハンドラに安全に処理を委託します。

不変データの束縛: 抽出した [タイトル, URL] の配列構造をクラス内部で厳格にカプセル化。表示インデックスと内部ポインタを同期させることで、動的描画の詰まりを完全に防いでいます。
